Template Syntax

A prefix template is a path string where segments (separated by /) can contain function placeholders. Placeholders are resolved before the S3 query is made.

segment1/{functionName(ref, arg1, arg2)}/segment3/
data/{everyNth(key, 0, 2)}/{key}/
logs/{date(yyyy/MM/dd, -1d)}/errors/
reports/{upper(everyNth(key, 0, 2))}/{key}/
Segments
  • A literal segment is used as-is: myprefix, 2024, ABCDEFGH
  • A placeholder is wrapped in { }: {everyNth(key, 0, 2)}
  • Use \{ to include a literal { in the path
References

Inside a function call, the first argument is a reference:

Reference Meaning
key The key value entered by the user
p1, p2, … pN The value of segment N in the template (1-based)

Rules for pN: A reference to a literal segment is always allowed at any position. A reference to a function segment is only allowed if N < current position (left-to-right resolution).

Function Chaining

Functions can be nested. The inner function is resolved first and its result is passed to the outer:

{upper(everyNth(key, 0, 2))} → everyNth result in uppercase
{lower(left(key, 5))} → first 5 chars in lowercase
{left(everyNth(key, 0, 2), 4)} → everyNth result, first 4 chars
{upper(everyNth(p3, 0, 2))} → everyNth on literal segment 3, uppercased
Wildcard

A * at the end of the Key field performs a prefix search. Only trailing wildcards are supported (S3 native prefix listing).

Key: ABCDE* → lists all objects whose key starts with ABCDE under the resolved prefix
